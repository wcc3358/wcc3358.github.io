# Dubbo方式使用方法

使用dubbo前提是您的系统已经集成了Dubbo，推荐性能要求较高的接口使用该方式，如果对实时性要求不严格的话，推荐在您的系统将结果做本地缓存处理。

# 要求

本系统为了不过多的耦合系统，不再采用之前的facade方式，也就要求各个系统自己提供相关的接口，有个好处是，这样的方式可以自定义接收的响应参数。

要求各系统必须将自己提供的接口放于系统路径为：com.changan.carbond.dubbo下，且名称、方法名、入参与dubbo方式API中提供的名称一致，其实就是要求方法签名一致，如com.changan.carbond.dubbo.IGlobalUserCarDubboService#getUserCarByTuid，入参为String类型的tuid。响应参数可以自定义，路径不要求，字段的个数、顺序不要求。
